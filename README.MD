# iCloud Photos Web Downloader

一个基于 **Playwright** 的非官方脚本，用来通过 **iCloud 网页端** 下载照片原片。

它会自动打开浏览器、进入 `icloud.com.cn`、协助进入“照片”页面，并从你上次下载的位置继续向后逐张下载。**如果是首次运行，没有下载过照片，脚本会等待你手动点开要下载的第一张照片，以此为起点继续向后逐张下载。** 下载完成后，脚本还会将文件时间还原为照片拍摄时间，并在遇到压缩包时自动解压出其中的多张照片。

> **注意**
>
> - 这是一个**非官方**项目，与 Apple 无关。
> - 仅建议用于**个人合法备份、归档或迁移自己的 iCloud 照片**。
> - 当前实现**仅支持 Windows**；如果在 macOS 或 Linux 上运行，脚本会直接退出。
> - 脚本依赖系统自带的 `tar` 处理压缩包；如果当前 Windows 环境里检测不到可用 `tar`，脚本会直接提示你使用 **Windows 10 或更高版本**。

> - 登录密码、双重验证、设备信任等步骤依然由你在网页中**手动完成**。


## 为什么会开发这个项目

iCloud 照片下载这件事，在很多社区里都被反复讨论；它不是一个“完全没人做”的问题，而是一个**大家都在找更顺手方案**的问题。这个项目的出发点并不是“为了重新发明一个下载器”，而是因为常见方案各有明显痛点：

- **在 iPhone 上同步到本地**：看起来最原生，但实际使用里经常会遇到同步卡住、原片未落地、空间不足、状态不透明等问题
- **直接在网页端批量下载**：操作直观，但大批量时容易报错或中断，而且下载后的文件往往不便于按拍摄时间继续整理和检索
- **通过 Privacy 页面申请数据包**：全量一次性导出，没法做增量下载；下载回来后也不方便按照片拍摄时间持续整理
- **使用成熟方案 `icloudpd`**：功能很强大，但对不少用户来说存在配置门槛

所以这个仓库想解决的是一个更具体、也更接地气的场景：

- 希望通过 **iCloud 网页端** 完成下载，而不是依赖手机同步
- 希望从某一张已知照片开始，**持续向后下载**，中断后还能继续
- 希望整个过程是**可见的，安全的**的，登录和验证步骤依然由你在浏览器里操作
- 希望下载后还能顺手处理一些细节，比如**重名避让、时间戳修正、ZIP 自动解压**

换句话说，这个仓库更像是一个**面向个人归档场景的定制化网页自动化脚本**：它不追求覆盖所有 iCloud 照片下载需求，而是想在“比纯手工省事”和“比完整工具链更轻量”之间，提供一个更容易上手、也更方便继续改造的中间方案。

## 这个项目如何工作的

这个仓库里的源码主要做了下面几件事：

- 自动打开 Chromium 浏览器并进入 `https://www.icloud.com.cn`
- 自动点击登录入口，并自动填入 Apple 账号邮箱
- 等待你手动输入密码并完成后续验证
- 进入 iCloud 首页后自动打开“照片”
- 从 `config.json` 中读取上次下载进度 `lastDownloadedUrl`，跳转到对应照片后，自动点击“下一张图像”
- 如果读不到上次下载进度，就按“首次运行”处理：等待你手动打开第一张要下载的照片，并自动把当前页面 URL 记为新的起点
- 自动打开“下载选项”，优先下载 **未修改的原片**，保存到本地目录，自动处理重名避免覆盖
- 尝试根据照片的拍摄时间设置文件时间
- 如果下载结果是压缩包，会自动解压并移动出其中的多张照片
- 每次成功下载后，自动更新 `config.json` 里的 `lastDownloadedUrl`，方便下次续传


## 如何运行

假设你已经具备可用的 `npm` / Node.js 环境，那么只需要准备以下内容：

- 一个可以登录 `https://www.icloud.com.cn` 的 Apple 账号邮箱
- 项目依赖安装完成（npm install）
- Playwright 所需浏览器安装完成（npx playwright install chromium）

进入项目目录后，执行：

```powershell
npm run download your_apple_id@example.com
```

## 配置文件

项目根目录下需要有一个 `config.json`，示例：

```json
{
  "savePath": "d:\\iCloud_webdownload\\",
  "downloadTimeoutSeconds": 60,
  "lastDownloadedUrl": "..."
}
```

### 配置项说明

- `savePath`
  - 下载文件保存目录
  - 如果目录不存在，脚本会自动创建
  - 如果不配置，默认会保存到项目目录下的 `download` 文件夹

- `downloadTimeoutSeconds`
  - 单个文件下载等待超时时间，单位是秒
  - 不配置时默认是 `60`

- `lastDownloadedUrl`
  - 上次下载到的照片页面链接
  - 如果不存在这个值；脚本会提示你在浏览器中手动点开要下载的第一张照片，并自动读取顶部 URL 作为起点
  - 如果已经存在该值，脚本启动后会先跳转到这个链接，再从那里继续向后下载

## 常见说明

### 1. 为什么不是全自动登录

当前实现会自动填邮箱，但**不会自动输入密码**。这是有意保留的设计：让你直接在 Apple 官方登录页面里输入密码，而不是把密码交给脚本、配置文件或其他中间环节，这样通常更让人安心。另一方面，iCloud 登录本身也经常伴随验证码、双重验证、设备确认等人工步骤，完全自动化并不稳定。

### 2. 为什么当前只支持 Windows

当前版本先只支持 Windows，一个直接的原因是：这个工具主要面向**普通用户**提供，而在这类用户里，**Windows 仍然是最常见的使用环境**。既然大多数用户本来就在 Windows 上使用，那么当前优先支持 Windows，会更贴近实际使用场景。

另一方面，如果这个脚本工具后面得到了更多用户认可、反馈也比较稳定，再继续支持 macOS 也不迟。等到那时，再针对 macOS 的系统命令、文件时间处理方式和运行体验做专门适配，会更稳妥一些。


### 3. ZIP 文件会怎么处理

如果 iCloud 返回的是压缩包，脚本会尝试调用系统 `tar` 进行解压，并把其中 `iCloud 照片` 目录里的文件移动到最终保存目录。

### 4. 下载中断怎么办

脚本每次成功下载后都会更新 `config.json` 里的 `lastDownloadedUrl`。重新启动后，可以从新的进度继续下载。
